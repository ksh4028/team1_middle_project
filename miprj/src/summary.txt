[프로젝트 보고서] RFP 분석을 위한 지능형 로컬 RAG 시스템 구축
1. 개발 목적 (Purpose)
보안 중심의 문서 분석 환경 구축: 제안요구서(RFP)와 같은 민감한 내부 데이터를 처리하기 위해 외부 API 의존도를 최소화하고, Ollama(Exaone 3.5) 기반의 로컬 LLM 환경을 우선적으로 구축.

정교한 기술적 의사결정 지원: 단순한 검색을 넘어 예산, 발주기관, 마감일 등 **수치 및 카테고리 데이터(Metadata)**를 정확히 필터링하여 사업 기회 분석의 정확도를 높임.

복합 질문 해결: "1억 이상 사업 중 IT 관련 공고를 찾아줘"와 같이 자연어와 조건이 섞인 질문을 SQL과 벡터 검색으로 나누어 처리하는 고도화된 검색 시스템 지향.

2. 개발 전략 (Strategy)
다단계 검색 아키텍처 (Advanced RAG Pipeline):

Query Transformation: Multi-Query 기능을 통해 사용자의 질문을 다각도로 확장하여 검색 누락 방지.

Hybrid Retrieval: 키워드 기반 BM25와 의미 기반 **FAISS(BGE-M3)**를 결합하고 RRF(Reciprocal Rank Fusion) 알고리즘으로 결과 최적화.

Intent-based SQL Filtering: midprj_sqlresult.py를 통해 질문에서 검색 의도를 추출, SQLite DB에서 정확한 메타데이터 필터링 수행.

2단계 재정렬 (Two-stage Reranking):

1차로 대량의 문서를 검색한 후, BGE-Reranker v2-m3 로컬 모델을 사용하여 문맥적 관련성이 가장 높은 순으로 재배치.

데이터 전처리 고도화: unstructured 및 pdfplumber를 활용하여 PDF 내 표와 텍스트를 정밀 추출하고, 요약 캐싱 시스템으로 속도 향상.

유연한 모델 선택 (LLM Selector): OpenAI와 Ollama(로컬)를 용도(추론, 요약, 평가)에 따라 선택적으로 사용할 수 있는 팩토리 패턴 적용.

3. 개발 경과 (Progress)
인프라 및 전처리 엔진 구축: HWP/PDF 파싱 유틸리티 및 SQLite 기반의 메타데이터/채팅 이력 관리 시스템 구축.

검색 성능 고도화: HybridRetriever와 DenseReranker를 연동하고, 임베딩 모델(BGE) 최적화를 통해 검색 품질 확보.

SQL 의도 분석기 구현: 자연어 질문에서 예산 범위, 기관명 등을 추출하여 SQL 쿼리로 변환하는 로직 완성.

평가 및 분석 시스템 도입: midprj_eval.py를 통해 Faithfulness(신뢰성), Relevance(관련성)를 측정하고, util_analyze_model.py로 파이프라인별 성능을 시각화하여 최적의 파라미터(K값, Temperature 등) 도출.

예외 처리 및 안정화: 계산 과정에서의 NaN 발생 문제 해결 및 토큰 제한(Context Window) 관리를 위한 History Truncation 로직 적용.

5. 회고 및 향후 과제 (Retrospective & Future Work)
회고:

기술적 성과: 로컬 환경에서도 BGE-Reranker와 Exaone 모델의 조합을 통해 상용 API(OpenAI)에 근접하는 답변 품질을 확보함.

데이터 정합성: 메타데이터 필터링과 벡터 검색의 결합으로 인해 정형/비정형 데이터가 섞인 RFP 문서 분석에서 강력한 성능을 발휘함.

향후 과제:

HWP 파싱 완벽화: 현재 PDF 변환 방식에서 나아가 한글(HWP) 문서의 복잡한 표 구조를 직접 해석하는 파서 고도화.

실시간 성능 최적화: Reranker와 LLM 추론 속도를 개선하기 위한 양자화(Quantization) 및 배치 처리 로직 도입.

멀티 에이전트 확장: 검색된 내용을 바탕으로 제안서 초안까지 작성해 주는 에이전트 기반 워크플로우 확장.
